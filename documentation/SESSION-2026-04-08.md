### Key Engineering Learnings
1. **Alpha grid search** was the biggest single improvement — learned alpha goes to 0.97 (too GNN-heavy), grid search finds 0.7-0.9 optimal per dataset
2. **IL-PCSR LR schedule** (warmup + 1.2x total steps) keeps minimum LR, prevents full decay
3. **DGL install**: conda Python 3.11 + `pip install torch==2.4.0 --index-url cu124` + `pip install dgl -f data.dgl.ai/wheels/torch-2.4/cu124/repo.html`. If graphbolt error: `echo "pass" > dgl/graphbolt/__init__.py`
4. **Output folders** must be method-specific (`adapted/`, `adapted_prox50/`, `full/`) to avoid overwriting

### Key Research Findings
1. **Vocabulary overlap is the only failure predictor** across 4 datasets — query length/structure don't matter
2. **Hard negatives score HIGHER than positives** on every similarity measure — precision problem, not recall
3. **3 legal perspectives** (domain, function, abstraction) are independent (ρ≈0) and discriminative
4. **Co-relevant statutes cluster** (median 18 apart, 85% same book) vs hard negatives (median 297 apart) — justifies proximity edges
5. **Optimal alpha correlates with lexical overlap**: less overlap → more GNN weight

### Papers Analyzed Critically
- **QDER** (SIGIR 2025): gold standard — proves channel independence (ρ=-0.022), per-component ablation
- **LegalMALR** (2026): 6 agents but no independence proof, expensive inference
- **SyLeR** (CIKM 2025): actually legal QA not retrieval, assumes syllogistic structure exists
- **G-DSR** (EACL 2023): statute graph is query-agnostic, only tested on BSARD
- **IL-PCSR** (EMNLP 2025): Para-GNN source, our baseline
